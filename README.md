#Behavioral Platform Divestment in Software Engineering: A Dynamic Multilevel StudyDescription#
This dataset contains objective digital trace telemetry tracking the week-by-week behavioral patterns of 1,752 tech-savvy software engineers across 9,331 worker-weeks. Extracted via the GitHub REST API, this data is designed to empirically measure the structural drivers and behavioral endpoints of digital stress within hyper-collaborative engineering ecosystems.The dataset operationalizes the core constructs of the Job Demands-Resources (JD-R) model and the Exit-Voice-Loyalty (EVL) framework. It provides the necessary variables to test a structural cascade: how a broader Digital Resource Portfolio (DRP)—measured by the count of actively managed public repositories—drives Information Overload (IO), which forces Context-Switching Attention Fragmentation (CAF). Ultimately, the data captures the behavioral endpoint of this cascade: Behavioral Platform Divestment (BPD). BPD is operationalized as a concurrent behavioral state characterized by communicative deterioration (voiced frustration) and the active deletion of digital assets (exit preparation).This dataset enables researchers to test dynamic multilevel models and examine how technical execution strain, specifically the Rework Intensity Ratio (RIR), operates as a parallel, additive demand alongside structural workflow fragmentation.Data DictionaryThe data_set.csv file contains the following columns:developer_id: Unique, anonymized identifier for each software engineer.week_num: The chronological week number of the observation period.control_tenure_days: Account Tenure. The total number of days since the user account was created (Standardized as a control variable).control_followers: Social Capital. The number of followers the developer has on the platform (Log-transformed as a control variable).eda_public_repos: Digital Resource Portfolio (DRP). The total number of active public repositories owned or managed by the developer.weekly_mentions_handled: The count of inbound IssueCommentEvent mentions directed at the developer during the week.weekly_prs_touched: The count of inbound PullRequestEvents the developer interacted with during the week.(Note: weekly_mentions_handled + weekly_prs_touched = Information Overload (IO) raw score)distinct_repos_touched_weekly: Context-Switching Attention Fragmentation (CAF). The count of distinct repositories contributed to within a single seven-day period.total_weekly_actions: Total Weekly Activity. The gross count of all recorded actions by the developer in the given week.RIR_ratio: Rework Intensity Ratio (RIR). The proportion of total weekly activity consumed by error-driven, corrective behavior (Calculated as (weekly_mentions_handled + 1) / (total_weekly_actions + 1)).BPD_composite: Behavioral Platform Divestment (BPD). A composite Z-score index averaging the log-transformed signals of communicative deterioration (negative sentiment in issue comments) and exit preparation (DeleteEvent frequency).MethodologyThis data was extracted via the GitHub REST API and aggregated into a weekly temporal granularity. All continuous predictor and moderator variables were mean-centered to prevent artificial multicollinearity. To account for the dynamic and cumulative nature of digital stress, dynamic multi-level modeling (MLM) was employed, estimating a three-equation system with one-week autoregressive lags ($t-1$) for all dependent variables.
